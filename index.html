The WebAssembly program can be compiled from multi-language and embeded in many edge devices with small footprint, high performance, memory safety, sandbox securty.

WasmEdge is responsible for ensuring its performance and securty and providing many exclusive extensions and APIs.

  • High-performance(based on LLVM AOT)
  • WASI-like Extensions
  • JavaScript Support
  • Cloud Native Management and Orchestration
  • Cross-platform Support
  • Easy Extensibility
  • Easy to Embed into a Host Application

Wasm is faster 100 times than Linux container at cold start-up time.

WasmEdge could be faster 10%-20% than native code.

  • Network sockets
  • Async polling
  • Tensorflow inference
  • Key value storage
  • Database connector
  • Gas meters for resource constraints
  • ES6 module and std API support
  • Implement JS APIs in Rust
  • Import C native shared library functions as JS function
  • CRI-O and Kubernetes compatibility
  • Sidecar apps in Kubernetes-based service meshes
  • Dapr microservices
  • Linux OSes dated back to 2010 for both x86 and ARM CPUs
  • Mac OS X for both x86 and m1 CPUs
  • Windows
  • Microkernel and RTOS

Build customized runtimes with native functions in C or GO

  • Embed WasmEdge functions in C, Go, Rust, Node.js and Python host applications
  • Embedded function runtime for service mesh proxies (e.g., proxy-wasm for Envoy and MOSN proxies)
